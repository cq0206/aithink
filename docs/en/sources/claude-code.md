# Claude Code Module

> This module groups Claude Code related materials in one place: architecture notes, visual explainers, systems analysis, and future implementation writeups.

## Why this deserves a dedicated module

Claude Code is better understood as an agent runtime than as a simple coding CLI.
It has a main loop, tool governance, multi-agent delegation, a safety layer, context management, and product-grade lifecycle handling.

That makes it worth curating as a standalone topic.

## Why `ccunpacked.dev` is useful

- It gives a fast visual overview of how Claude Code works.
- The `Agent Loop` animation is especially good for building intuition.
- It pairs well with longer architecture writeups and source-level reading.

## Why `harness-books` is useful

- It frames Claude Code as part of a broader harness design problem.
- One book focuses on Claude Code itself.
- The second compares Claude Code and Codex as two different control architectures.
- We currently index these as Chinese in-site reading guides instead of mirroring the full text.
